# TEE Evidence Verification

## Overview

Chutes runs workloads inside Intel TDX (Trust Domain Extensions) confidential VMs. NVIDIA GPUs are attached via passthrough with hardware-enforced memory isolation, ensuring the host cannot access either VM or GPU memory. GPU attestation is integrated into the overall TEE verification process. Chutes verifies all servers and chute instances on its network as part of normal operations. The evidence endpoints documented here exist so that **third parties can independently verify TEE integrity without having to trust Chutes.**

A caller supplies a cryptographic nonce and receives attestation evidence proving that a specific instance is running on genuine TEE hardware. This evidence can be verified directly against Intel and NVIDIA verification infrastructure.

Additionally, Chutes supports **end-to-end client-side encryption (E2E)**, where each instance generates an ML-KEM 768 (post-quantum) keypair at startup inside the TEE. The instance's E2E public key is cryptographically bound to the attestation evidence, allowing callers to verify that the key they are encrypting to was generated inside a genuine TEE -- not by a man-in-the-middle.

### How It Works

1. The caller generates a random nonce (32 bytes, hex-encoded).
2. The caller requests evidence from the Chutes API, passing the nonce.
3. Chutes retrieves a fresh TDX quote and GPU attestation evidence from the instance, bound to the caller's nonce and the instance's E2E public key.
4. The caller receives the evidence and verifies it independently:
  - **Intel TDX quote** -- verified via Intel's DCAP (Data Center Attestation Primitives) infrastructure, proving the workload runs in a genuine TDX enclave.
  - **NVIDIA GPU evidence** -- verified via the NVIDIA Attestation SDK, proving the GPUs are genuine NVIDIA hardware isolated from the host.
5. The caller verifies that the quote's `report_data` contains `SHA256(nonce + e2e_pubkey)`, which proves both freshness (nonce) and that the E2E public key was generated inside the attested TEE.

## Key Concepts

**Nonce** -- A 64 hex character (32 byte) freshness token generated by the caller. Combined with the instance's E2E public key and hashed into the TDX quote's `report_data` field and the GPU evidence, preventing replay attacks. Must be cryptographically random.

**TDX Quote** -- An Intel TDX attestation quote that cryptographically proves the workload runs inside a genuine TDX enclave. Contains hardware measurements (MRTD, RTMRs), a `report_data` field, and a signature verifiable via Intel DCAP. The first 64 hex characters of `report_data` contain `SHA256(nonce + e2e_pubkey)`, binding both the caller's nonce and the instance's encryption key to the hardware attestation.

**GPU Evidence** -- Per-GPU NVIDIA attestation evidence proving the GPUs are genuine NVIDIA hardware isolated from the host. Verified via the NVIDIA Remote Attestation SDK.

**E2E Public Key** -- An ML-KEM 768 (post-quantum) public key generated by the instance at startup inside the TEE. Used for end-to-end encryption of requests and responses. Because it is hashed into the quote's `report_data`, a successful attestation proves this key was generated inside the attested TEE environment.

**Certificate** -- The TLS certificate (base64-encoded DER) of the server hosting the instance, included in the response for reference.

**Dynamic Instances** -- Chutes scales instances up and down based on demand. New instances can appear at any time and existing instances can be removed. It is the caller's responsibility to periodically re-verify and to check any new instance IDs they encounter. See [Dynamic Instances and Caller Responsibilities](#dynamic-instances-and-caller-responsibilities) for details.

## API Reference

### Get E2E Instances

Discover E2E-capable instances for a chute and retrieve their public keys and invocation nonces.

```
GET /e2e/instances/{chute_id}
```

**Path Parameters**


| Parameter  | Type   | Description |
| ---------- | ------ | ----------- |
| `chute_id` | string | Chute ID    |


**Authentication**

API key required.

**Rate Limit**

10 requests per minute.

**Response**

```json
{
  "instances": [
    {
      "instance_id": "<instance-id>",
      "e2e_pubkey": "<base64-encoded ML-KEM 768 public key>",
      "nonces": ["<nonce-token>", "..."]
    }
  ],
  "nonce_expires_in": 60,
  "nonce_expires_at": 1740000000
}
```


| Field                     | Type    | Description                                                                                               |
| ------------------------- | ------- | --------------------------------------------------------------------------------------------------------- |
| `instances`               | array   | E2E-capable instances with their public keys.                                                             |
| `instances[].instance_id` | string  | Instance ID.                                                                                              |
| `instances[].e2e_pubkey`  | string  | Base64-encoded ML-KEM 768 public key for E2E encryption. This key is generated inside the TEE at startup. |
| `instances[].nonces`      | array   | Single-use nonces for E2E invocation (not the same as attestation nonces).                                |
| `nonce_expires_in`        | integer | Seconds until the invocation nonces expire.                                                               |
| `nonce_expires_at`        | integer | Unix timestamp when the invocation nonces expire.                                                         |

### Get Evidence for a Chute (All Instances)

Retrieves TEE evidence for every running instance of a chute in a single call.

```
GET /chutes/{chute_id_or_name}/evidence?nonce={nonce}
```

**Path Parameters**


| Parameter          | Type   | Description             |
| ------------------ | ------ | ----------------------- |
| `chute_id_or_name` | string | Chute ID (UUID) or name |


**Query Parameters**


| Parameter | Type   | Required | Description                                                     |
| --------- | ------ | -------- | --------------------------------------------------------------- |
| `nonce`   | string | Yes      | 64 hex characters (32 bytes). Must be cryptographically random. |


**Authentication**

An API key is required unless the chute is public. For public chutes, no authentication is needed.

**Rate Limit**

60 requests per minute.

**Response**

```json
{
  "evidence": [
    {
      "quote": "<base64-encoded TDX quote>",
      "gpu_evidence": [
        { "...per-GPU attestation fields..." }
      ],
      "instance_id": "<instance-id>",
      "certificate": "<base64-encoded DER certificate>"
    }
  ],
  "failed_instance_ids": ["<instance-id>"]
}
```


| Field                     | Type   | Description                                                                                                                    |
| ------------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------ |
| `evidence`                | array  | List of `TeeInstanceEvidence` objects, one per running instance.                                                               |
| `evidence[].quote`        | string | Base64-encoded Intel TDX quote for this instance.                                                                              |
| `evidence[].gpu_evidence` | array  | List of per-GPU evidence dicts. Each dict contains the GPU's attestation evidence with base64-encoded fields where applicable. |
| `evidence[].instance_id`  | string | The instance ID this evidence belongs to.                                                                                      |
| `evidence[].certificate`  | string | Base64-encoded DER format TLS certificate from the server.                                                                     |
| `failed_instance_ids`     | array  | Instance IDs where evidence retrieval failed (instances exist but the fetch encountered a transient error).                    |


**Error Responses**


| Status | Description                                                                           |
| ------ | ------------------------------------------------------------------------------------- |
| 400    | Invalid nonce format (must be exactly 64 hex characters) or chute is not TEE-enabled. |
| 404    | Chute not found or you do not have access.                                            |
| 429    | Rate limit exceeded.                                                                  |
| 502    | Attestation service unavailable. The attestation proxy could not be reached.          |

**Rate Limit**

60 requests per minute.

**Response**

```json

  "quote": "<base64-encoded TDX quote>",
  "gpu_evidence": [
```

| Field          | Type   | Description                                                                 |

## Verification Guide

This section walks through how to independently verify the evidence returned by these endpoints.

### Step 1: Generate a Nonce

Generate a cryptographically random 32-byte nonce, hex-encoded to 64 characters:

```python
import secrets

nonce = secrets.token_hex(32)  # 64 hex characters
```

Or from the command line:

```bash
openssl rand -hex 32
```

### Step 2: Get Instance Public Keys

Retrieve the E2E public keys for the chute's instances. You will need these to verify the attestation evidence.

```python
import requests

API_KEY = "your-api-key"
CHUTE_ID = "your-chute-id"

e2e_response = requests.get(
    f"https://api.chutes.ai/e2e/instances/{CHUTE_ID}",
    headers={"Authorization": f"Bearer {API_KEY}"},
)
if not e2e_response.ok:
    print(f"Error {e2e_response.status_code}: {e2e_response.text}")
    raise SystemExit(1)
e2e_data = e2e_response.json()

# Build a map of instance_id -> e2e_pubkey
pubkeys = {
    inst["instance_id"]: inst["e2e_pubkey"]
    for inst in e2e_data["instances"]
}
```

### Step 3: Request Evidence

**Using curl:**

```bash
NONCE=$(openssl rand -hex 32)

curl "https://api.chutes.ai/chutes/{chute_id_or_name}/evidence?nonce=$NONCE" \
  -H "Authorization: Bearer $CHUTES_API_KEY"
```

**Using Python:**

```python
evidence_response = requests.get(
    f"https://api.chutes.ai/chutes/{CHUTE_ID}/evidence",
    params={"nonce": nonce},
    headers={"Authorization": f"Bearer {API_KEY}"},
)
if not evidence_response.ok:
    print(f"Error {evidence_response.status_code}: {evidence_response.text}")
    raise SystemExit(1)
data = evidence_response.json()
```

### Step 4: Verify the Intel TDX Quote

The TDX quote proves the workload is running inside a genuine Intel TDX enclave. Verification uses the `dcap-qvl` Python library, which validates the quote's cryptographic signature against Intel's DCAP collateral (certificate chain, TCB info, etc.).

```python
import base64
import hashlib
import json
from dcap_qvl import get_collateral_and_verify

quote_b64 = data["evidence"][0]["quote"]  # or data["quote"] for single instance
quote_bytes = base64.b64decode(quote_b64)

# Verify the quote signature against Intel's DCAP collateral.
# This contacts Intel's Provisioning Certification Service to fetch
# the collateral needed for verification.
verified_report = await get_collateral_and_verify(quote_bytes)
report = json.loads(verified_report.to_json())
```

After verification, check the following:

**1. Signature status must be `UpToDate`:**

```python
status = report.get("status", "Unknown")
assert status == "UpToDate", f"Quote verification status: {status}"
```

**2. Debug mode must be disabled:**

The TD must not be running in debug mode. Debug mode is indicated by bit 0 of `td_attributes`:

```python
td_report = report.get("report", {}).get("TD10", {})
td_attributes = td_report.get("td_attributes", "")
attr_value = int(td_attributes, 16)
debug_enabled = bool(attr_value & 1)
assert not debug_enabled, "TD is running in debug mode"
```

**3. Verify the nonce and E2E public key binding:**

The instance computes `SHA256(nonce + e2e_pubkey)` and embeds this in the first 64 hex characters of the quote's `report_data`. This binds both the caller's nonce (freshness) and the instance's E2E public key (identity) to the hardware attestation.

```python
instance_id = data["evidence"][0]["instance_id"]
e2e_pubkey = pubkeys[instance_id]

# Compute expected report_data: SHA256 of nonce concatenated with the E2E public key
expected_report_data = hashlib.sha256((nonce + e2e_pubkey).encode()).hexdigest().lower()

# Extract report_data from the raw quote bytes
# TD report starts at byte offset 48; report_data is at offset 520-584 within the TD report
td_report_bytes = quote_bytes[48:632]
report_data_hex = td_report_bytes[520:584].hex().lower()
extracted_report_data = report_data_hex[:64]

assert extracted_report_data == expected_report_data, (
    "report_data mismatch -- the nonce or E2E public key does not match the attestation"
)
```

This check is critical because it proves:

- The attestation is fresh (your nonce is included in the hash).
- The E2E public key was generated inside the attested TEE (the key is included in the hash that the TEE signed).
- No one has substituted a different public key (any change to the key would change the hash).

### Step 5: Verify NVIDIA GPU Evidence

The GPU evidence proves the instance is using genuine NVIDIA GPUs that are isolated from the host. Verification uses the `nv-attestation-sdk` package, which validates the evidence against NVIDIA's Remote Attestation Service.

The GPU evidence is also bound to `SHA256(nonce + e2e_pubkey)`, the same value used in the TDX quote's `report_data`:

```python
from nv_attestation_sdk.attestation import Attestation, Devices, Environment

gpu_evidence = data["evidence"][0]["gpu_evidence"]  # or data["gpu_evidence"] for single instance

client = Attestation()
client.add_verifier(Devices.GPU, Environment.REMOTE, "", "")
client.set_nonce(expected_report_data)
result = client.attest(gpu_evidence)
assert result, "GPU attestation failed"
```

This verifies that:

- The GPUs are genuine NVIDIA hardware, isolated from the host.
- The evidence is bound to the same nonce + public key hash, ensuring consistency with the TDX attestation.

## End-to-End Encryption (E2E)

Chutes supports end-to-end client-side encryption where prompts and responses are encrypted directly between the client and a specific GPU instance. No intermediary -- not the Chutes API, not the load balancers, not the miner -- can see the plaintext.

### How E2E Works

1. **Instance discovery** -- `GET /e2e/instances/{chute_id}` returns E2E-capable instances with their ML-KEM 768 public keys and single-use invocation nonces.
2. **Attestation** -- The caller verifies the instance's TEE evidence (as described above). The E2E public key is cryptographically bound to the attestation via `SHA256(nonce + e2e_pubkey)` in `report_data`, proving the key was generated inside the TEE.
3. **Client-side encryption** -- The caller encrypts the request using the instance's ML-KEM 768 public key. Only the instance can decrypt it.
4. **Opaque relay** -- The Chutes API relays the encrypted blob. It only sees ciphertext.
5. **Instance-side decryption** -- The instance decrypts with its ephemeral private key (generated at startup inside the TEE), processes the request, and encrypts the response back.

### Post-Quantum Key Exchange

Each instance generates an ML-KEM 768 keypair at startup -- a NIST-standardized post-quantum key encapsulation mechanism. Even future quantum computers cannot retroactively decrypt captured traffic.

### Why Attestation Matters for E2E

Without attestation, a man-in-the-middle could substitute their own public key for the instance's real key. By verifying that `SHA256(nonce + e2e_pubkey)` appears in the TDX quote's `report_data`, the caller confirms the public key was generated inside the attested TEE. This closes the loop: the caller knows they are encrypting to a key that only the genuine TEE instance holds the private key for.

### Client Library

The [chutes-e2ee-transport](https://github.com/chutesai/chutes-e2ee-transport) package provides an OpenAI-compatible client plugin. Drop it in and your existing code gains full E2E encryption with no changes beyond initialization. Alternatively, [e2ee-proxy](https://github.com/chutesai/e2ee-proxy) offers a simple proxy-based option for E2E encryption.

## Complete Example

A full end-to-end script that discovers instances, requests evidence, and verifies both the TDX quote (with E2E public key binding) and GPU attestation.

**Usage:**

```bash
python verify_chute.py --api-key YOUR_API_KEY --chute-id YOUR_CHUTE_ID
```

**Script:**

```python
import argparse
import asyncio
import base64
import hashlib
import json
import secrets
import sys

import requests
from dcap_qvl import get_collateral_and_verify
from nv_attestation_sdk.attestation import Attestation, Devices, Environment

API_BASE = "https://api.chutes.ai"


def verify_tdx_quote(
    quote_b64: str, expected_nonce: str, e2e_pubkey: str
) -> dict:
    """Verify a TDX quote and return the parsed report."""
    quote_bytes = base64.b64decode(quote_b64)

    # Verify signature against Intel DCAP collateral
    verified_report = asyncio.get_event_loop().run_until_complete(
        get_collateral_and_verify(quote_bytes)
    )
    report = json.loads(verified_report.to_json())

    # Check signature status
    status = report.get("status", "Unknown")
    if status != "UpToDate":
        raise ValueError(f"TDX quote verification status: {status}")

    # Check debug mode is disabled
    td_report = report.get("report", {}).get("TD10", {})
    td_attributes = td_report.get("td_attributes", "")
    if td_attributes:
        attr_value = int(td_attributes, 16)
        if attr_value & 1:
            raise ValueError("TD is running in debug mode")

    # Verify nonce + E2E public key binding in report_data
    expected_report_data = (
        hashlib.sha256((expected_nonce + e2e_pubkey).encode()).hexdigest().lower()
    )
    td_report_bytes = quote_bytes[48:632]
    report_data_hex = td_report_bytes[520:584].hex().lower()
    extracted = report_data_hex[:64]
    if extracted != expected_report_data:
        raise ValueError(
            "report_data mismatch: nonce + E2E public key does not match attestation"
        )

    return report, expected_report_data


def verify_gpu_evidence(
    gpu_evidence: list, expected_report_data: str
) -> bool:
    """Verify NVIDIA GPU attestation evidence."""
    client = Attestation()
    client.add_verifier(Devices.GPU, Environment.REMOTE, "", "")
    client.set_nonce(expected_report_data)
    return client.attest(gpu_evidence)


def main():
    parser = argparse.ArgumentParser(description="Verify TEE attestation for a Chutes chute")
    parser.add_argument("--api-key", required=True, help="Chutes API key")
    parser.add_argument("--chute-id", required=True, help="Chute ID to verify")
    args = parser.parse_args()

    api_key = args.api_key
    chute_id = args.chute_id

    # 1. Generate a fresh nonce
    nonce = secrets.token_hex(32)
    print(f"Generated nonce: {nonce}")

    # 2. Get E2E instance public keys
    e2e_response = requests.get(
        f"{API_BASE}/e2e/instances/{chute_id}",
        headers={"Authorization": f"Bearer {api_key}"},
    )
    if not e2e_response.ok:
        print(f"Error fetching E2E instances: {e2e_response.status_code} {e2e_response.text}")
        sys.exit(1)
    pubkeys = {
        inst["instance_id"]: inst["e2e_pubkey"]
        for inst in e2e_response.json()["instances"]
    }
    print(f"Found {len(pubkeys)} E2E-capable instance(s)")

    # 3. Request evidence
    evidence_response = requests.get(
        f"{API_BASE}/chutes/{chute_id}/evidence",
        params={"nonce": nonce},
        headers={"Authorization": f"Bearer {api_key}"},
    )
    if not evidence_response.ok:
        print(f"Error fetching evidence: {evidence_response.status_code} {evidence_response.text}")
        sys.exit(1)
    chute_evidence = evidence_response.json()

    print(f"Received evidence for {len(chute_evidence['evidence'])} instance(s)")
    if chute_evidence["failed_instance_ids"]:
        print(
            f"Warning: evidence retrieval failed for instances: "
            f"{chute_evidence['failed_instance_ids']}"
        )

    # 4. Verify each instance
    all_passed = True
    for instance_evidence in chute_evidence["evidence"]:
        instance_id = instance_evidence["instance_id"]
        print(f"\nVerifying instance: {instance_id}")

        e2e_pubkey = pubkeys.get(instance_id)
        if not e2e_pubkey:
            print(f"  Skipped: no E2E public key found for this instance")
            all_passed = False
            continue

        # Verify TDX quote (includes nonce + pubkey binding)
        try:
            report, expected_report_data = verify_tdx_quote(
                instance_evidence["quote"], nonce, e2e_pubkey
            )
            print(f"  TDX quote: PASSED (status={report['status']})")
        except Exception as e:
            print(f"  TDX quote: FAILED ({e})")
            all_passed = False
            continue

        # Verify GPU evidence
        try:
            gpu_ok = verify_gpu_evidence(
                instance_evidence["gpu_evidence"], expected_report_data
            )
            if gpu_ok:
                print(f"  GPU attestation: PASSED")
            else:
                print(f"  GPU attestation: FAILED")
                all_passed = False
        except Exception as e:
            print(f"  GPU attestation: FAILED ({e})")
            all_passed = False

    sys.exit(0 if all_passed else 1)


if __name__ == "__main__":
    main()
```

## Dynamic Instances and Caller Responsibilities

Chutes instances are ephemeral -- they scale up and down based on demand. This has important implications for verification:

- **The chute evidence endpoint returns evidence for all currently-running instances.** The set of instances can change between calls.
- **New instances may appear at any time.** An instance that was not present in a previous evidence response may appear in the next one.
- **Previously-verified instances may disappear.** An instance you verified earlier may have been scaled down and no longer exists.
- **Callers should re-verify periodically.** Generate a fresh nonce and request new evidence on a schedule appropriate for your trust requirements.
- **Verify any new instance IDs.** When the evidence response contains an `instance_id` you haven't seen before, verify its evidence before trusting it.
- **Handle `failed_instance_ids` appropriately.** This field lists instances that exist but whose evidence could not be fetched due to a transient error (e.g., the attestation proxy was temporarily unreachable). These instances are still running but could not be verified in this request. Consider retrying or treating them as unverified depending on your requirements.

## What You're Verifying


| Check                                         | What It Proves                                                                                                                          |
| --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| Intel TDX quote signature (DCAP)              | The workload is running inside a genuine Intel TDX enclave on real hardware.                                                            |
| `SHA256(nonce + e2e_pubkey)` in `report_data` | The attestation is fresh (nonce) and the E2E public key was generated inside the attested TEE (not substituted by a man-in-the-middle). |
| Debug mode disabled                           | The TD is running in production mode, not a debug configuration that would weaken isolation.                                            |
| NVIDIA GPU attestation                        | The GPUs are genuine NVIDIA hardware, isolated from the host (the host cannot access GPU memory).                                       |
